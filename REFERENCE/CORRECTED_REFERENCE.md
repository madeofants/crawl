## Complete System Architecture

### Character Creation Menu Flow

```
Main Menu
├── Game Mode Selection
│   └── choose_game (NORMAL, SPRINT, TUTORIAL, etc.)
│
├── Species Selection
│   ├── Menu shows all available species
│   ├── Filtered by game mode restrictions
│   └── Recommended/unrecommended marks
│
├── Job Selection
│   ├── Menu shows available jobs for selected species
│   ├── Some jobs restricted for some species
│   └── Sprint mode has additional restrictions
│
├── Weapon Selection
│   ├── Menu shows available weapons for job
│   ├── Some jobs get restricted weapon lists
│   └── Some jobs can choose any weapon
│
└── Name Entry & Game Start
    ├── Player enters name (or random name generated)
    └── Game initialization begins
```

## Data Structures

### newgame_def (File: newgame-def.h, 73 lines)

This struct holds all character creation choices:

```cpp
struct newgame_def
{
    string name;                // Character name
    game_type type;            // Game mode (NORMAL, SPRINT, etc.)
    uint32_t seed;             // Random seed
    bool pregenerate;          // Pre-generate dungeon?
    species_type species;      // Selected species
    job_type job;              // Selected job
    weapon_type weapon;        // Selected weapon
    bool fully_random;         // Fully random character?
};
```

### species_def (File: species-def.h)

Each species has these properties:

```cpp
struct species_def
{
    species_type species;           // Enum: SP_HUMAN, SP_ELF, etc.
    const char* name;              // Display name: "Human", "Elf"
    const char* abbrev;            // 2-letter code: "Hu", "El"
    int xp_mod;                    // XP multiplier (affects leveling speed)
    int hp_mod;                    // HP bonus (in tenths)
    int mp_mod;                    // MP bonus (in tenths)
    int wl_mod;                    // Willpower modifier
    vector<mutation_type> mutations; // Starting mutations
    species_flags flags;           // Special properties (draconian, size, etc.)
    habitat_type habitat;          // Where can live (water, lava, etc.)
    undead_state_type undeadness;  // If undead, what type
    size_type size;                // Body size (TINY to GIANT)
};
```

### job_def (File: job-def.h, 34 lines)

Each job has these properties:

```cpp
struct job_def
{
    const char* name;                    // Display name: "Warrior", "Wizard"
    const char* abbrev;                  // 2-letter code: "Wa", "Wz"
    int s, i, d;                        // Starting Strength, Intelligence, Dexterity
    vector<species_type> recommended_species;  // Which species are good here
    vector<spell_type> library;         // Starting spells
    vector<string> equipment;           // Starting items (vault format)
    weapon_choice wchoice;              // Weapon selection: none/plain/good
    vector<pair<skill_type, int>> skills; // Starting skills (e.g., Fighting:3)
};
```

## Restriction System (File: ng-restr.h, 21 lines)

The system enforces which species can be which jobs:

```cpp
enum char_choice_restriction
{
    CC_UNRESTRICTED,  // No restriction (was wrongly documented as CC_ALLOWED)
    CC_RESTRICTED,    // Not recommended, but playable
    CC_BANNED,        // Cannot play this combination
};
```

### How Restrictions Work

1. **Check restricting function:** `char_class_allowed(species_type sp, job_type job)`
2. **Check recommending function:** `job_recommends_species(job_type job, species_type sp)`
3. **Sprint mode special rule:** `_job_allowed()` wrapper at `newgame.cc:141-145` bans JOB_DELVER in Sprint

## Menu System

### Classes Involved (File: menu.h, outer-menu.h, newgame.cc)

**UINewGameMenu** (newgame.cc:1167-1300)
- Central UI controller for character creation
- Manages two OuterMenu instances:
  - `main_items` - Main menu options
  - `sub_items` - Description and details
- Uses Switcher widget to show descriptions

**OuterMenu** (outer-menu.h, 111 lines)
- Hierarchical menu display
- Shows game modes, species, jobs, weapons
- Filters based on restrictions
- Provides descriptions for each item

**MenuEntry** (menu.h, 500+ lines)
- Individual menu item with:
  - `text` - Display text
  - `tags` - Semantic tags (VIABLE, RESTRICTED, etc.)
  - `on_select` - Callback function
  - `hotkeys` - Keyboard shortcuts
  - `quantity` - Item count
  - `enabled` - Selectable or grayed out
  - `tile` - Visual representation

## Game Initialization (File: ng-setup.cc, 574 lines)

After character choices are made:

### Main Function: `_setup_generic()`

This is called after all character choices and does:
1. Initialize player stats based on species/job
2. Give starting spells from job library
3. Give starting equipment from job equipment list
4. Set up dungeon branches (Dungeon, Lair, Depths, etc.)
5. Generate initial dungeon level
6. Place player at starting position

### Key Functions in ng-setup.cc

- `setup_game()` - Main orchestrator
- `_setup_generic()` - Core initialization
- `_give_items_skills()` - Give starting equipment
- `_init_player()` - Initialize player character
- `_setup_tutorial()` - Tutorial-specific initialization

- `dat/jobs/` - Job data definitions

### Build Integration
1. Run `make` in `crawl-ref/source/`
2. Build system runs Python scripts
3. Scripts generate .h files
4. C++ compilation proceeds with generated headers
