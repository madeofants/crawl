## Architecture

```
User (Prose Input)
    ↓
AI Narrator (Prose Prompt Generator & Action Converter)
    ↓
Game State Bridge (Read state, send commands)
    ↓
DCSS Engine (Simulation, combat, mechanics, dungeon)
    ↓
Database (Game context, NPC dialogue, custom content)
```

## Why We're Modifying DCSS

We need to:
- Control character creation programmatically (our Narrator will create starting characters)
- Create custom game modes optimized for narrative gameplay
- Access game state to feed to the Narrator
- Execute commands from code (not player input)
- Potentially add custom content/items/species for thematic variety

Traditional roguelike UI is completely bypassed - player sees only prose.
